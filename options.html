Contact Form Configuration Options

Contact form is very customizable. You way want to:
  • Change the look and feel
  • Add fields
  • Restrict the referrer
  • Allow international characters
  • Turn off client side verification

Change the Look and Feel

You probably want to make the contact form feel like it is really part of the rest of your site. You may want your standard color scheme, links to other pages, and so on. To do this you can change the $input_page_head, $input_page_body_start, and $input_page_body_end.

$input_page_head

This variable gets inserted into the head section of the html document. The default has the page title (Contact Us) and some style sheet information for errors, size of text boxes, and the color of the required indicator.

$input_page_body_start

The default only contains an h1 tag with "Contact Us". Links that are at the top or sides of all the pages on your site would go in this variable.

$input_page_body_end

The default contains the required field footnote and a link to the Contact Form website. This would be a good place for copyright notices and other footers.

Add Fields

Contact form comes with standard name, address, subject and message fields. It is very easy to add additional fields. When you add additional field you need to add them to the
  1. %Form_Fields - a regular expression to validate the field. There are plenty of prebuilt regular expressions to choose from:
    • $SOMETHING - A generic required field (multiple line textarea)
    • $ANYTHING - A generic optional field (multiple line textarea)
    • $ONE_LINE_REQUIRED - A generic required field (single line text field)
    • $ONE_LINE_OPTIONAL - A generic optional field (single line text field)
    • $ZIPCODE_REQUIRED - Required postal code.
    • $ZIPCODE_OPTIONAL - Optional postal code.
    • $PHONE_REQUIRED - Requride phone nmuber.
    • $PHONE_OPTIONAL - Optional phone number.
    • $EMAIL_REQUIRED - For a required email address
    • $EMAIL_OPTIONAL - For an optional email address
    • $PRICE_REQUIRED - A required price (number with two decimal points)
    • $PRICE_OPTIONAL - An optional price (number with two decimal points)
    • FLOAT_REQUIRED - A number possibly with decimal points
    • $FLOAT_OPTIONAL - A number possibly with decimal points (optional)
    • $INTEGER_REQUIRED - A number without decimal points
    • $INTEGER_OPTIONAL - A number without decimal points (optional)
    • If you know regular expressions you can make up your own
  2. @Field_Order - To control the order in which they appear in the form.
  3. %Error_Messages - To specify the message for when the user enters something invalid.
  4. %Form_Type - text, textarea, or hidden to determine how the field will look on the form.
  5. %Field_Descriptions - For text describes the field on the form.

Restrict the Referrer

Changing this will prevent other sites from linking to your contact form. You would want to consider this especially if some other site sets up a button that allows users to click the button to immediatly send you a prefilled email - effectively spamming you with the same message over and over.

Allow International Characters

It would be possible to change the character set used in contact form from Latin 1 (ISO-8859-1) to unicode (UTF-8) or something else that can handle characters in a wider variety of languages. To do so simply modify the my $charset variable.

Turn Off Client Side Verification

Setting "my $use_client_side_verification = 0;" will turn off the javascript check in the browser that makes sure the form elements are valid before submisson. Contact Form also does a server side check, so this is not needed (in fact some browsers can't handle it and Contact Form doesn't make them do the check). You might want to turn this off if you don't want users to be able to look at the source of the page and figure out what patterns you actually would accept for certain fields.
